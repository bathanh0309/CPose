# fixbackend.md — Kế hoạch sửa backend để UI CPose thật sự nối được với runtime

Tài liệu này dùng để đưa cho Claude/Codex sửa **backend trước**, theo hướng thực dụng, bám sát code hiện có của CPose.

Mục tiêu của tài liệu:
- chỉ ra đúng các **điểm đứt kết nối** giữa `index.html/app.js/registration.js` và Flask backend
- đưa ra **contract API + Socket.IO thống nhất**
- cung cấp **code mẫu đủ chi tiết** để Claude bám vào sửa
- ưu tiên làm cho demo **chạy được end-to-end** trước, không refactor lớn

---

## 0. Mục tiêu sửa backend

Sau khi sửa xong backend, hệ thống phải đạt được tối thiểu các hành vi sau:

1. Frontend được serve từ Flask ở `http://localhost:5000`, không mở file HTML rời.
2. Load được `resources.txt` và trả về danh sách camera.
3. Bật được preview RTSP bằng snapshot endpoint.
4. Chạy được recorder Phase 1 và emit event khi có short clip.
5. Chạy được recognizer Phase 3 ở 2 mode:
   - `rtsp`
   - `multicam_folder`
6. Original / Processed view lấy được ảnh từ backend qua snapshot endpoint.
7. Video queue bên phải lấy được dữ liệu thật từ backend hoặc socket event.
8. Registration UI gọi được backend, có `next_id`, `start`, `stop`, `snapshot`.
9. Bảng trái nhận event log summary.
10. Bảng đen giữa nhận metric log runtime (`fps/frame/conf/adl/event`).

---

## 1. Chẩn đoán gốc — vì sao UI đang “chỉ là giao diện”

### 1.1 Frontend và backend chưa cùng runtime contract
Hiện backend có các nhóm endpoint:
- `/api/config/*`
- `/api/recording/*`
- `/api/analysis/*`
- `/api/pose/*`
- `/api/registration/*`

Nhưng UI lại đang mong rất nhiều thứ cùng lúc:
- queue clip
- metrics realtime
- original/processed snapshot
- registration overlay
- multicam folder + RTSP mode

=> Cần **một contract thống nhất** để frontend biết gọi đúng endpoint nào cho từng mode.

### 1.2 `browser folder` khác với `server folder`
Frontend chọn folder `.mp4` từ browser **không đồng nghĩa** backend có quyền dùng path cục bộ đó.

**Kết luận:**
- Demo ổn định nhất: backend tự đọc `data/multicam/` server-side.
- Upload folder từ browser chỉ là tính năng mở rộng, không phải đường chạy chính trước mắt.

### 1.3 `/api/pose/start` hiện có nguy cơ lỗi logic
Trong code hiện tại, `start_pose()` đang dùng `body.get(...)` nhưng thiếu dòng parse body ở đầu hàm trong snapshot bạn đang bám theo. Đây là lỗi phải sửa ngay.

### 1.4 Socket event names chưa được chốt
Frontend cần các event dạng:
- `rec_status`
- `camera_status`
- `clip_saved`
- `pose_progress`
- `registration_progress`
- `registration_done`
- `event_log`
- `metric_log`

Nhưng backend hiện mới có một phần nhỏ helpers trong `ws_handlers.py`. Phải chuẩn hóa event contract.

### 1.5 Registration UI và registration backend có thể đang dùng hai nguồn camera khác nhau
- Frontend preview webcam bằng `getUserMedia()`
- Backend registration service lại có thể đọc camera local riêng của server process

=> Nếu không thống nhất source, UI thấy webcam nhưng backend không chụp từ đúng nguồn đó.

**Chiến lược demo an toàn:**
- Cho registration flow backend làm nguồn chuẩn.
- Frontend chỉ mở webcam để preview nếu muốn, nhưng dữ liệu đăng ký phải đến từ backend session.

```python
# app/bootstrap/app_factory.py
import os
import logging
from pathlib import Path
from flask import Flask, send_from_directory
from flask_cors import CORS
from flask_socketio import SocketIO

from app.utils.config_schema import AppConfig
from .logging_setup import setup_logging

logger = logging.getLogger("[AppFactory]")
socketio = SocketIO()


def create_app(static_dir: Path, config: AppConfig) -> Flask:
    setup_logging(config.logging)
    logger.info("Initializing CPose application factory...")

    flask_app = Flask(
        __name__,
        static_folder=str(static_dir),
        static_url_path="/static",
    )

    flask_app.config["SECRET_KEY"] = os.environ.get("SECRET_KEY", "cpose-dev-secret")

    # Tăng limit để tránh fail khi upload video / folder demo
    flask_app.config["MAX_CONTENT_LENGTH"] = 512 * 1024 * 1024  # 512 MB

    CORS(flask_app, resources={r"/*": {"origins": config.server.cors_origins}})

    from app.services.recorder import RecorderManager
    from app.services.recognizer.orchestrator import RecognizerService

    recognizer = RecognizerService(config, socketio)
    recognizer.start()

    recorder = RecorderManager()

    flask_app.config["recorder"] = recorder
    flask_app.config["recognizer"] = recognizer

    def on_clip_ready(clip_path: Path, cam_id: str):
        logger.info("Auto-processing trigger: %s from %s", clip_path.name, cam_id)
        recognizer.enqueue_clip(cam_id, Path(clip_path))

    flask_app.config["on_clip_ready_cb"] = on_clip_ready

    _register_blueprints(flask_app)
    _register_ui_routes(flask_app, static_dir)

    socketio.init_app(
        flask_app,
        cors_allowed_origins=config.server.cors_origins,
        async_mode=config.server.socket_async_mode,
    )

    _register_socket_handlers()
    return flask_app


def _register_blueprints(app: Flask):
    from app.api import api_bp
    app.register_blueprint(api_bp)


def _register_ui_routes(app: Flask, static_dir: Path):
    @app.get("/")
    def dashboard():
        return send_from_directory(str(static_dir), "index.html")


def _register_socket_handlers():
    try:
        import app.api.ws_handlers  # noqa: F401
    except Exception:
        logger.exception("Failed to register SocketIO handlers.")
```

## 4. Sửa `app/api/ws_handlers.py` để chuẩn hóa event runtime

## 4.1 Mục tiêu
Tạo một nơi emit event thống nhất để services gọi.

## 4.2 Code mẫu

```python
# app/api/ws_handlers.py
from __future__ import annotations
import logging
import threading
from typing import Any
from app import socketio

logger = logging.getLogger("[WS]")
_pending: dict[str, dict[str, Any]] = {}


def emit_event_log(event: str, cam: str = "--"):
    payload = {
        "time": _now_time_string(),
        "event": event,
        "cam": cam,
    }
    socketio.emit("event_log", payload)


def emit_metric_log(
    *,
    cam: str = "--",
    fps: float | int | None = None,
    frame: str | int | None = None,
    conf: float | None = None,
    adl: str | None = None,
    event: str = "runtime",
):
    payload = {
        "time": _now_time_string(),
        "cam": cam,
        "fps": fps if fps is not None else "--",
        "frame": frame if frame is not None else "--",
        "conf": conf if conf is not None else "--",
        "adl": adl or "--",
        "event": event,
    }
    socketio.emit("metric_log", payload)


def emit_camera_status(cam_id: str, *, fps=None, frame=None, conf=None, status="IDLE"):
    socketio.emit(
        "camera_status",
        {
            "cam_id": cam_id,
            "fps": fps,
            "frame": frame,
            "conf": conf,
            "status": status,
        },
    )


def emit_rec_status(*, is_recording: bool, cam_id: str | None = None):
    socketio.emit(
        "rec_status",
        {
            "is_recording": is_recording,
            "cam_id": cam_id,
        },
    )


def emit_clip_saved(*, filename: str, cam_id: str, path: str, preview_url: str | None = None):
    socketio.emit(
        "clip_saved",
        {
            "filename": filename,
            "cam_id": cam_id,
            "path": path,
            "preview_url": preview_url,
        },
    )


def emit_pose_progress(
    *,
    cam_id: str,
    clip_stem: str,
    frame_id: int,
    total_frames: int,
    fps: float | None,
    conf: float | None,
    adl: str | None,
    pct: float | int | None,
    event: str = "pose_progress",
):
    socketio.emit(
        "pose_progress",
        {
            "cam_id": cam_id,
            "clip": clip_stem,
            "frame_id": frame_id,
            "total_frames": total_frames,
            "fps": fps,
            "conf": conf,
            "adl": adl,
            "pct": pct,
            "event": event,
        },
    )


def request_face_registration(clip_stem: str, cam_id: str, timeout: float | None = 300.0) -> dict | None:
    logger.info("Requesting face registration for %s (%s)", clip_stem, cam_id)

    ev = threading.Event()
    _pending[clip_stem] = {"event": ev, "payload": None}

    try:
        socketio.emit("register_face_request", {"clip_stem": clip_stem, "cam": cam_id})
    except Exception as exc:
        logger.warning("Failed to emit register_face_request: %s", exc)

    waited = ev.wait(timeout=timeout)
    entry = _pending.pop(clip_stem, None)
    if not waited or entry is None:
        logger.warning("Registration wait timed out or no response for %s", clip_stem)
        return None

    return entry.get("payload")


@socketio.on("connect")
def _handle_connect():
    logger.info("UI connected via SocketIO")


@socketio.on("disconnect")
def _handle_disconnect():
    logger.info("UI disconnected from SocketIO")


@socketio.on("register_face_done")
def _handle_register_face_done(data):
    clip_stem = str((data or {}).get("clip_stem", ""))
    if not clip_stem:
        logger.warning("register_face_done missing clip_stem")
        return

    entry = _pending.get(clip_stem)
    if not entry:
        logger.warning("No pending registration found for %s", clip_stem)
        return

    entry["payload"] = data
    entry["event"].set()


def _now_time_string() -> str:
    import datetime as _dt
    return _dt.datetime.now().strftime("%H:%M:%S")
```

```python
@api_bp.route("/workspace/upload_multicam_clips", methods=["POST"])
def upload_multicam_clips():
    files = request.files.getlist("files")
    if not files:
        return _error("No files uploaded", 400)

    target_dir = _app_module.BASE_DIR / "data" / "multicam_uploads"
    target_dir.mkdir(parents=True, exist_ok=True)

    saved = []
    for f in files:
        if not (f.filename or "").lower().endswith(".mp4"):

        filename = secure_filename(f.filename)
        dst = target_dir / filename
        f.save(dst)
        saved.append(dst)

    if not saved:
        return _error("No valid .mp4 files uploaded", 400)

    clips = sort_multicam_clips(saved)
    return _ok(
        mode="multicam_folder",
        total=len(clips),
        clips=[
            {
                "name": p.name,
                "path": str(p),
                "url": f"/api/video/{p.relative_to(_app_module.DATA_DIR).as_posix()}" if p.is_relative_to(_app_module.DATA_DIR) else None,
            }
            for p in clips
        ],
```

```python
@api_bp.route("/recording/start", methods=["POST"])
def start_recording():
    body = _body_json()
    selected_cams = body.get("cameras") or []

    if isinstance(selected_cams, str):
        selected_cams = [selected_cams]

    all_cams = _storage.parse_resources(_app_module.RESOURCES_FILE)
    cameras = []

    if len(selected_cams) > 0 and isinstance(selected_cams[0], str):
        for c in all_cams:
            if c.get("cam_id") in selected_cams or c.get("label") in selected_cams:
                cameras.append(c)
    else:
        cameras = selected_cams

    if not cameras:
        return _error("No valid cameras found or specified", 400)

    if get_recorder().is_running():
        return _error("Recording is already running", 409)

    try:
        storage_limit_gb = float(body.get("storage_limit_gb", _app_module.DEFAULT_STORAGE_LIMIT_GB))
    except (TypeError, ValueError):
        return _error("storage_limit_gb must be numeric", 400)

    get_recorder().start(
        cameras=cameras,
        storage_limit_gb=storage_limit_gb,
        output_dir=_app_module.RAW_VIDEOS_DIR,
        model_path=_app_module.MODEL_PHASE1,
        config=body.get("phase1_config", {}),
        on_clip_ready=current_app.config.get("on_clip_ready_cb"),
    )

    from app.api.ws_handlers import emit_event_log, emit_rec_status
    emit_event_log("Recorder started", "SYS")
    emit_rec_status(is_recording=True)

    return _ok(message="Recording started", cameras=len(cameras))
```

```python
@api_bp.route("/pose/start", methods=["POST"])
def start_pose():
    body = _body_json()
    mode = str(body.get("mode", "multicam_folder")).strip().lower()
    recognizer = get_pose()

    if mode == "rtsp":
        cameras_req = body.get("cameras") or []
        all_cams = _storage.parse_resources(_app_module.RESOURCES_FILE)
        selected_cams = [c for c in all_cams if c.get("cam_id") in cameras_req]
        if not selected_cams:
            return _error("No valid cameras for RTSP mode", 400)

        if not get_recorder().is_running():
            get_recorder().start(
                cameras=selected_cams,
                storage_limit_gb=float(body.get("storage_limit_gb", _app_module.DEFAULT_STORAGE_LIMIT_GB)),
                output_dir=_app_module.RAW_VIDEOS_DIR,
                model_path=_app_module.MODEL_PHASE1,
                on_clip_ready=current_app.config.get("on_clip_ready_cb"),
            )

        from app.api.ws_handlers import emit_event_log
        emit_event_log("Pose pipeline started in RTSP mode", "SYS")
        return _ok(status="started", mode="rtsp", cameras=len(selected_cams))

    if mode == "multicam_folder":
        folder = body.get("folder")
        if folder:
            video_dir = _resolve_dir(folder, _app_module.RAW_VIDEOS_DIR)
        else:
            # fallback mặc định cho demo server-side
            video_dir = _app_module.BASE_DIR / "data" / "multicam"

        if not video_dir.is_dir():
            return _error(f"Directory not found: {video_dir}", 400)

        clips = sort_multicam_clips(video_dir.glob("*.mp4"))
        if not clips:
            return _error("No .mp4 files found in the selected folder", 400)

        for clip in clips:
            cam_id = _app_module.extract_multicam_camera_id(clip) if hasattr(_app_module, "extract_multicam_camera_id") else "unknown"
            recognizer.enqueue_clip(cam_id, clip)

        from app.api.ws_handlers import emit_event_log
        emit_event_log(f"Pose pipeline started in multicam_folder mode ({len(clips)} clips)", "SYS")
        return _ok(status="started", mode="multicam_folder", total_clips=len(clips))

    return _error(f"Unsupported mode: {mode}", 400)
```

```python
        "adl": status.get("adl", "unknown"),
        "lamp_state": lamp_state,
        "pending_results": status.get("pending_results", []),
    }
    return jsonify(normalized)
```

## 5.11 Registration endpoints — giữ nhưng sửa nhẹ payload

```python
@api_bp.route("/registration/start", methods=["POST"])
def start_registration():
    body = _body_json()
    source = body.get("source")
    rtsp_url = body.get("rtsp_url")
    user_name = body.get("name")
    user_age = body.get("age", "??")
    person_id = body.get("person_id") or _registration.get_next_id()

    if not user_name:
        return _error("Name is required", 400)

    if source == "rtsp" and not rtsp_url:
        return _error("RTSP URL is required for RTSP source", 400)

    camera_source = "local" if source == "local" else rtsp_url

    def on_progress(data):
        from app.api.ws_handlers import emit_metric_log
        _on_socket_emit("registration_progress", data)
        emit_metric_log(cam="REG", event=data.get("message", "registration_progress"))

    def on_done(data):
        _on_socket_emit("registration_done", data)
        if data.get("status") == "success":
            get_pose().refresh_face_database()

    session_id = _registration.start_session(
        source=camera_source,
        user_name=user_name,
        user_age=user_age,
        person_id=person_id,
        on_progress=on_progress,
        on_done=on_done,
    )

    return _ok(session_id=session_id, status="started", user_id=person_id, person_id=person_id)
```

## 6. Sửa service layer để emit đúng dữ liệu cho UI

> Phần này không cần refactor mạnh. Chỉ cần thêm hooks emit.

## 6.1 Recorder service cần emit gì

Khi recorder chạy:
- emit `rec_status`
- emit `camera_status`
- khi có short clip -> emit `clip_saved`
- emit `event_log`

### Pseudocode
```python
# trong recorder loop
emit_camera_status(cam_id, fps=fps, frame=frame_idx, conf=best_conf, status="ACTIVE")

if started_recording:
    emit_rec_status(is_recording=True, cam_id=cam_id)
    emit_event_log(f"Recording started on {cam_id}", cam_id)

if clip_saved:
    emit_clip_saved(
        filename=clip_path.name,
        cam_id=cam_id,
        path=str(clip_path),
        preview_url=f"/api/video/raw_videos/{clip_path.name}",
    )
    emit_event_log(f"Short clip saved: {clip_path.name}", cam_id)
```

## 6.2 Recognizer service cần emit gì

Khi recognizer xử lý clip:
- cập nhật snapshot original/processed buffer
- emit `pose_progress`
- emit `metric_log`
- cập nhật `get_status()`
- stage pending result để UI bên phải thấy clip đã xử lý xong

### Pseudocode
```python
# trong recognizer processing loop
self._status["running"] = True
self._status["mode"] = mode
self._status["current_clip"] = clip_stem
self._status["current_cam"] = cam_id
self._status["current_frame"] = frame_idx
self._status["total_frames"] = total_frames
self._status["fps"] = fps
self._status["conf"] = conf
self._status["adl"] = adl_label

self._latest_original_jpeg = encode_jpeg(original_frame)
self._latest_processed_jpeg = encode_jpeg(processed_frame)

emit_pose_progress(
    cam_id=cam_id,
    clip_stem=clip_stem,
    frame_id=frame_idx,
    total_frames=total_frames,
    fps=fps,
    conf=conf,
    adl=adl_label,
    pct=round(frame_idx * 100 / max(total_frames, 1), 1),
    event="pose_progress",
)
```

### Hết clip
```python
self._pending_results[clip_stem] = {
    "clip_stem": clip_stem,
    "preview_video": str(preview_path),
    "temp_dir": str(clip_output_dir),
    "saved": False,
}
emit_event_log(f"Finished pose processing: {clip_stem}", cam_id)
```

## 6.3 Registration service cần chắc 2 việc
- `get_next_id()` phải tăng dần thật
- `get_snapshot(session_id)` nên trả ảnh có overlay bbox/text nếu có thể

### Gợi ý code mẫu cho ID tăng dần
```python
class RegistrationManager:
    def __init__(self):
        self._next_id = 1
        self._sessions = {}

    def get_next_id(self) -> str:
        return f"{self._next_id:04d}"

    def reserve_next_id(self) -> str:
        value = f"{self._next_id:04d}"
        self._next_id += 1
        return value
```

> Nếu đang có persistence/database rồi, nên lấy max id từ storage thay vì chỉ giữ trong RAM.
